## 1. Notes (padho · 20 min)

### SDLC Phases (yaad order)
1. **Requirement** — kya banana hai (SRS)
2. **Design** — kaise banana hai (HLD/LLD)
3. **Implementation / Coding**
4. **Testing**
5. **Deployment**
6. **Maintenance**

### Models (exam favourites)
| Model | Idea |
|---|---|
| **Waterfall** | Sequential, ek phase ke baad next; change mushkil |
| **Agile** | Iterative, sprints, customer feedback |
| **Spiral** | Risk analysis + iterations |
| **V-Model** | Verification & Validation — testing har stage pe map |
| **Prototype** | Pehle sample, phir final |

### Testing Types
| Type | Meaning |
|---|---|
| **Unit** | Smallest module test |
| **Integration** | Modules together |
| **System** | Full system |
| **Acceptance (UAT)** | User/client accept |
| **Regression** | Bug fix ke baad old features re-test |
| **Black-box** | Bina code dekhe (input→output) |
| **White-box** | Code logic dekh ke |

### SE Quick Terms
- **SRS** = Software Requirements Specification
- **Debugging** = bug find + fix
- **Coupling** = modules kitne dependent (low coupling better)
- **Cohesion** = module andar related (high cohesion better)
- **Version control** = Git etc. (track changes)
