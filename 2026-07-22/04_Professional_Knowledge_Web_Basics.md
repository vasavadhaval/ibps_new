## 1. Notes (padho · 20 min)

### Web stack (simple)
| Layer | Kaam | Example |
|---|---|---|
| **HTML** | Structure / content | headings, forms, links |
| **CSS** | Style / look | colors, layout |
| **JavaScript** | Behaviour / logic | clicks, validation |
| **HTTP/HTTPS** | Browser ↔ server communication | request/response |

### HTTP basics
- **HTTP** = HyperText Transfer Protocol (port **80**)
- **HTTPS** = secure HTTP with encryption (port **443**)
- Methods: **GET** (fetch), **POST** (send/submit), PUT, DELETE
- Stateless protocol — har request alag (session/cookie se state manage)

### Status codes (yaad)
| Code | Meaning |
|---|---|
| **200** | OK |
| **301/302** | Redirect |
| **400** | Bad Request |
| **401** | Unauthorized |
| **403** | Forbidden |
| **404** | Not Found |
| **500** | Internal Server Error |

### JSON
- **JavaScript Object Notation** — data exchange format
- Light-weight, key:value pairs
- APIs me common: `{ "name": "Ram", "age": 25 }`

### Other exam terms
- **URL** = address of resource
- **DNS** = domain → IP (port 53)
- **Cookie** = small data stored in browser (session/login)
- **Cache** = temporary store for speed
- **API** = interface between systems
- **Client–Server** = browser (client) requests, server responds
- **Responsive design** = mobile/desktop dono pe fit
